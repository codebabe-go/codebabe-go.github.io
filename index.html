常见问题

  1. string不能聚合怎么办?
    • 将该字段fielddata: true
    • 用keyword/text代替
  2. 聚合后列表展示条数只能显示10条?
    • es java api支持设置大小, 但是默认是10
  3. es中update的概念
    • es可以不设置id, 但是update需要有相对应的id, 如果没有update操作可以不设置id
  4. 在高并发写下可能会有VersionConflictEngineException的插入异常
    • 设置重试次数retryOnConflict

建议

  • java bean最好和es的字段名称保持一致, 因为es底层保存都是用json, 这样可以直接转换

jdk提供的指令

发表于 2017-08-08

jps

usage:

options 描述
-q 只输出LVMID, 省略主类的名称
-l 输出主类的全名, 如果进程执行的是jar包, 输出jar路径
-m 输出虚拟机进程启动时传递给主类main（）函数的参数
-v 启动虚拟机进程时候的参数

jstat

假设你已经在你要查的服务器上了, 查看gc情况, usage

觉得只要理解这个指令就可以了, vmid虚拟进程的id, interval多久监控改一次, times次数

看到42234这个虚拟进程(基于jps 输出的vmid), 最近20次每次10毫秒的gc情况

造成442234这个进程上一次fgc的原因, 适合排查问题

jmap

显示堆中对象统计情况, 包括类, 实例对象, 合集容量 通过这个指令可以看出哪个类(实例化对象)现在占的资源比较多, 来排查java代码是否有不合理实例化的现象

jstack

usage:

options 描述
-F 当正常信息不被响应的时, 强制输出线程堆栈
-l 除堆栈外, 显示关于锁的附加信息 常用
-m 调用本地方法时, 输出c/c++的堆栈信息
